# 企业智慧 OA 管理系统小组分工表

## 一、项目基本信息

项目名称：企业智慧 OA 管理系统

项目类型：JavaEE 企业级开发课程考核项目

项目架构：前后端分离 + SpringBoot 微服务架构

小组人数：6 人

人员配置：

- 前端开发：2 人
- 后端开发：3 人
- PPT / 文档 / 测试 / 汇报：1 人

技术栈：

- 后端：SpringBoot 3.x、Spring Cloud Alibaba、Nacos、MySQL、MyBatis / MyBatis-Plus、Redis、JWT、Maven
- 前端：Vue3、Element Plus、Vue Router、Pinia、Axios、ECharts
- 工程能力：统一返回结果、统一异常处理、网关鉴权、跨域处理、Nacos 配置中心、数据库脚本、接口文档、项目演示录屏

## 二、总体分工原则

为了保证 6 名小组成员可以并行开发，本项目按照“基础架构、权限模块、业务模块、前端框架、前端业务、文档汇报”进行拆分。

每名成员独立负责一个清晰的功能范围，尽量避免多人频繁修改同一批文件。

后端 1 先完成公共基础能力和网关搭建，后端 2、后端 3 在统一规范下分别开发独立业务服务。前端 1 负责前端基础框架、登录和权限页面，前端 2 负责业务页面和数据大屏。PPT / 文档负责人从项目开始同步整理文档、SQL、接口测试记录和答辩材料。

## 五、后端目录分工建议

```text
backend/
  oa-common/              后端 1 负责
  oa-gateway/             后端 1 负责
  user-service/           后端 2 负责
  attendance-service/     后端 3 负责
  flow-service/           后端 3 负责
  notice-service/         后端 3 负责
  report-service/         后端 3 或 PPT / 文档负责人协助
```

后端 1 需要优先完成 `oa-common` 和 `oa-gateway`，否则其他服务无法统一接入鉴权、返回格式和服务路由。

后端 2 聚焦用户权限体系，避免与业务服务混在一起。

后端 3 聚焦 OA 核心业务，包括考勤、审批、公告。三个服务之间业务关联较强，由同一名后端负责更容易保证逻辑一致。

## 八、建议选择的拓展功能

建议优先选择“数据可视化大屏 + 月度考勤报表导出”作为拓展功能。

原因：

- 实现成本相对可控
- 前后端都能体现工作量
- 答辩展示效果直观
- 与 OA 业务贴合度高
- 不会明显影响基础功能开发进度

可视化大屏建议包含：

- 部门人数统计
- 今日打卡人数统计
- 月度迟到 / 早退 / 旷工统计
- 请假 / 加班申请数量统计
- 待办 / 已办审批数量统计
- 公告阅读率统计

## 九、并行开发流程建议

### 第 1 天：搭框架

- 后端 1：搭建后端父工程、公共模块、网关服务
- 后端 2：设计用户权限表结构，创建 `user-service`
- 后端 3：设计考勤、审批、公告表结构，创建业务服务
- 前端 1：搭建 Vue3 项目、完成登录页和 Layout
- 前端 2：搭建业务页面路由和静态页面
- PPT / 文档负责人：整理需求分析、数据库初稿、分工表

### 第 2-3 天：完成基础接口和页面

- 后端 1：完善 JWT、跨域、网关鉴权、Nacos 配置
- 后端 2：完成用户权限模块主要接口
- 后端 3：完成考勤、审批、公告主要接口
- 前端 1：完成系统管理相关页面
- 前端 2：完成考勤、审批、公告业务页面
- PPT / 文档负责人：同步整理接口文档和测试用例

### 第 4-5 天：联调和拓展功能

- 后端：修复联调问题，补充统计接口和报表导出
- 前端：对接真实接口，完善交互和页面美化
- PPT / 文档负责人：完成 README、接口测试记录、数据库说明

### 第 6-7 天：测试、录屏、答辩准备

- 全员：完整跑通登录、权限、考勤、审批、公告、大屏
- 后端：保证服务能稳定启动
- 前端：保证演示页面无明显 UI 问题
- PPT / 文档负责人：完成 PPT、演示录屏、答辩讲稿

## 十一、注意事项

- 每个人都需要有明确代码或文档贡献，方便答辩时说明个人工作量。
- 后端接口命名、返回格式、错误码需要统一，避免前端联调混乱。
- 前端页面可以先用 Mock 数据开发，等后端接口稳定后再切换真实接口。
- PPT / 文档负责人不要等项目最后一天才开始整理材料，应从第一天开始同步维护文档。
- 答辩前必须提前启动所有服务，不能现场临时启动项目。
- 演示前需要准备一套完整测试数据，包括管理员账号、普通员工账号、审批人账号、考勤记录、审批单据和公告数据。
