# 课程考核要求

## 一、课程考核基本信息

- **课程名称**：JavaEE 企业级开发
- **考核项目**：分布式微服务 OA 办公管理系统（前后端分离架构）
- **开发周期**：一周集中开发 + 答辩评审（暂定周六下午）
- **分组规则**：全班分为 9 个开发小组，每组 5~6 人，小组全员必须参与编码、文档、测试、答辩全流程，无实际贡献者考核不合格。
- **考核形式**：项目成果交付 + 现场答辩评审
- **培养衔接目标**：项目贴合企业真实开发标准，同步夯实相关工程实践能力

## 二、技术栈规范

### 2.1 基本技术栈

#### 后端技术栈（SpringBoot3 微服务体系）

- **核心框架**：SpringBoot 3.x + Spring Cloud Alibaba
- **微服务底座**：Nacos（注册中心 + 配置中心）
- **持久层**：MySQL + MyBatis + MyBatis-Plus（可自由选择）
- **缓存中间件**：Redis（分布式缓存、分布式锁）
- **身份认证**：JWT 无状态会话鉴权（网关统一鉴权，摒弃传统 Session）
- **构建工具**：Maven 多模块父工程管理

#### 前端技术栈

- **基础框架**：Vue3 + Element Plus
- **硬性要求**：UI 界面美观、交互流畅（前端颜值作为加分项）
- **接口调试**：统一对接后端网关 RESTful 接口，前后端完全分离（前后端分离架构）

## 三、项目功能考核要求

项目功能考核分为基础必做要求和拓展选做要求。

### 3.1 基础必做功能

在第二部分要求的技术栈基础上，完整实现 OA 系统基础模块核心业务功能。

#### 用户与组织权限模块（user-service）

- 员工账号登录、JWT 令牌签发、退出登录（无状态鉴权）
- 部门、岗位、员工信息 CRUD 管理

#### RBAC 权限模型

- 角色分配
- 菜单权限
- 接口权限控制

#### 考勤打卡核心模块（attendance-service）

- 员工上下班打卡接口
- 当日打卡记录存储
- 个人打卡记录查询

#### 简易审批模块（flow 基础版）

- 请假申请、加班申请单据提交
- 直属领导一级审批（同意 / 驳回操作）
- 待办、已办审批列表查询

#### 公告通知模块（notice-service）

- 管理员发布公司公告
- 员工查看已读 / 未读状态

#### 配套基础能力

- 全局统一异常处理、统一返回结果封装
- Nacos 配置中心托管业务参数（如打卡锁过期时间、迟到阈值，支持动态刷新）
- 跨域处理、全局 Token 拦截校验
- 数据库脚本、完整项目注释、基础操作日志

### 3.2 选做拓展功能

可根据自己的理解添加扩展功能，必须至少选择一项扩展功能。功能越丰富、越贴近企业场景，项目分数越高。可向老师进行需求评审，获取功能建议。

以下功能可作为扩展参考。

#### 分布式高并发优化

- 定时任务 XXL-Job 自动核算每日考勤（凌晨批量统计迟到、旷工、工时）
- MQ 异步消息推送：审批完成、考勤异常站内通知
- Sentinel 熔断降级，服务故障隔离，防止系统雪崩

#### 工作流进阶

- 集成 Flowable 工作流引擎，多级审批、转办、抄送、流程撤回

#### 分布式事务场景

- 员工离职联动多服务：人事档案归档、账号禁用、资产回收（Seata AT 实现）

#### 数据与检索优化

- Elasticsearch 实现公告、审批单据全文检索
- 月度考勤报表统计、Excel 导出

#### 附加业务模块

- 人事管理：入职、转正、调岗、劳动合同管理
- 资产管理：办公用品申领、固定资产管理

#### 工程化能力

- Docker 容器打包部署文档
- SkyWalking 链路追踪、全接口日志收集
- 接口文档 Knife4j、单元测试编写

#### AI 智能能力模块

贴合 OA 办公业务，后端对接大模型 API（通义千问 / 文心一言 / 本地轻量 AI 模型），开发智能办公能力。例如：

1. **智能审批助手**：自动解析请假、加班单据，智能判断请假合理性、风险提示；自动生成审批意见摘要。
2. **考勤智能分析**：AI 分析员工月度打卡数据，自动识别高频迟到、异常旷工员工，生成风险预警。
3. **智能办公问答**：内置 OA 知识库，员工可对话查询考勤规则、审批流程、人事制度。

#### 大数据可视化大屏模块

基于 Vue3 + DataV / ECharts 开发企业 OA 数据驾驶舱，整合全业务数据实时可视化。例如：

1. **人事数据看板**：各部门人数、岗位分布、入职 / 离职趋势柱状图、饼图。
2. **考勤数据大屏**：月度迟到 / 早退 / 旷工统计、部门工时对比、打卡热力图。
3. **审批流转看板**：待办 / 已办单据数量、审批平均耗时、各类申请占比环形图。
4. **数据实时刷新**：定时拉取后端统计接口，大屏自动更新图表数据。

## 四、小组分工规范

- **后端负责**：微服务架构搭建、Nacos / Gateway 网关配置、分布式公共组件封装（Redis、JWT 工具类）
- **后端开发**：拆分独立微服务模块（用户服务、考勤服务、审批服务、消息服务），完成 CRUD、业务逻辑、分布式锁 / 事务实现
- **前端负责人**：Vue3 项目搭建、全局路由、权限拦截、页面 UI 美化（保证前端颜值）
- **测试 & 文档负责人**：接口测试、数据库设计、项目文档撰写、答辩 PPT 制作

**要求**：每人独立负责至少一个业务模块，代码提交记录将作为个人贡献佐证。

## 六、答辩考核流程

### 6.1 小组汇报

- **架构讲解**：微服务拆分、Nacos、JWT 无状态鉴权、分布式解决方案
- **功能演示**：完整演示全部基础功能，展示选择的拓展功能
- **分工说明**：每人讲解自己负责模块、核心代码实现

### 6.2 教师提问

提问范围包括：

- 课上学习到的内容
- 分布式理论
- 微服务组件原理
- Redis 锁
- JWT 无状态会话
- Nacos 配置 / 注册中心
- 项目业务难点
- 代码细节
- 计算机应知的知识、面试题等

### 6.3 汇报准备要求

汇报之前，要提前把项目运行起来，不能到了讲解的时候再花时间等待项目启动。
