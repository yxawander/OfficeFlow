# OfficeFlow 考勤与薪资结算精细化规则明细表

> **文件版本**：v1.2 (全维度按工时精细化结算版)  
> **适用模块**：`attendance-service` 考勤服务 / `report-service` 报表服务 / 薪资结算模块

---

## 一、 核心计薪基准 (Base Salary Metrics)

为确保薪资计算符合国家劳动法规范与行业通用标准，系统采用如下基准：

| 计薪参数 | 标准设定值 | 备注说明 |
| :--- | :--- | :--- |
| **法定月计薪天数** | `21.75` 天/月 | 国家人社部规定月计薪天数 |
| **标准每日工时** | `8.0` 小时/天 | 标准工时制每日工作时长 |
| **标准日薪 (Daily Rate)** | `基本工资 / 21.75` | 精确保留 4 位小数，结算时四舍五入保留 2 位 |
| **标准时薪 (Hourly Rate)** | `标准日薪 / 8.0` = `基本工资 / 174` | 用于按小时/分钟精确折算加班费与离岗扣款 |

---

## 二、 薪资结算核心公式 (Final Salary Equation)

$$\text{实发工资 (Actual Salary)} = \text{基本工资} + \text{岗位补贴} + \text{加班费} - \text{离岗扣款(迟到/早退/缺卡)} - \text{旷工扣款} - \text{请假扣款}$$

> [!NOTE]
> 如果员工当月应扣除费用大于应发总额，实发工资在系统中自动进行保底处理，最小输出为 `0.00` 元。

---

## 三、 各薪资细项计算规则明细表 (Calculation Rules)

| 项目名称 | 关联考勤状态 / 数据源 | 算力与算法规则 | 惩罚/加发系数 | 核心数学公式 |
| :--- | :--- | :--- | :--- | :--- |
| **1. 基本工资** | `sys_user_salary.base_salary` | 员工签约月固定基本薪资 | `1.0` (全额) | `base_salary` |
| **2. 岗位补贴** | `sys_user_salary.allowance` | 交通补、餐补、岗位津贴等合集 | `1.0` (全额) | `allowance` |
| **3. 加班工资** | `flow_apply` (OVERTIME) + `attendance_record` | 取审批工时与实际打卡工时的最小值：$\min(\text{审批}, \text{实打})$ | `1.5` 倍时薪 | `overtimeHours * hourlyRate * 1.5` |
| **4. 迟到扣款** | `attendance_record.late_minutes` | 累计当月所有打卡记录中的迟到总分钟数 | `1.0` 倍时薪 | `(lateMinutes / 60.0) * hourlyRate` |
| **5. 早退扣款** | `attendance_record.early_leave_minutes` | 累计当月所有打卡记录中的早退总分钟数 | `1.0` 倍时薪 | `(earlyLeaveMinutes / 60.0) * hourlyRate` |
| **6. 迟到且早退** | `status = 'LATE_AND_EARLY'` | 单条记录同时含有迟到与早退分钟数，精确自动累加 | `1.0` 倍时薪 | `((lateMinutes + earlyLeaveMinutes) / 60.0) * hourlyRate` |
| **7. 未补卡缺卡** | `status = 'MISSING_CARD'` (月底未补卡) | 月底结算时仍未通过补卡流程更正的缺卡，每次折算 4.0 小时离岗 | `1.0` 倍时薪 | `(missingCardCount * 4.0) * hourlyRate` |
| **8. 旷工扣款** | `status = 'ABSENT'` | 全天无故缺勤或迟到超过旷工门槛 (4小时) | `2.0` 倍日薪 | `absentCount * dailyRate * 2.0` |
| **9. 请假扣款** | `flow_apply` (LEAVE) | 审批通过的事假/病假等扣薪假期天数 | `1.0` 倍日薪 | `leaveDays * dailyRate * 1.0` |

## 四、 状态更正与免扣规则 (Correction & Waiver Rules)

1. **补卡成功更正 (`RECHECKED`)**：
   * 员工针对“迟到”、“早退”、“既迟到又早退”、“缺卡”提交补卡申请；
   * 主管审批同意后，系统自动将状态转换为 `RECHECKED` (已补卡)，并更新打卡时间；
   * 结算时该日期更正为正常到勤，**完全免除离岗扣款**。

2. **请假成功更正 (`ON_LEAVE`)**：
   * 员工提前提交请假申请且主管审批同意；
   * 后端自动向考勤表写入 `ON_LEAVE` 状态；
   * 结算时仅按标准请假天数计算 1.0 倍日薪扣款，**免除 2.0 倍旷工处罚金**。

## 五、 实操全场景试算案例 (Practical Test Case)

### 📌 员工档案信息
* **姓名**：张伟
* **月基本工资**：￥17,400.00
* **岗位津贴**：￥1,000.00
* **标准日薪** = $17,400 / 21.75 = \mathbf{800.00 \text{ 元/天}}$
* **标准时薪** = $800 / 8.0 = \mathbf{100.00 \text{ 元/小时}}$

### 📌 当月考勤发生明细
1. 迟到 1 次（30 分钟），早退 1 次（30 分钟）；
2. 既迟到又早退 1 次（迟到 30 分钟 + 早退 30 分钟）；
   * **离岗总分钟** = $30 + 30 + (30 + 30) = 120 \text{ 分钟} = \mathbf{2.0 \text{ 小时}}$
3. 缺卡 1 次（月底未补卡） = $\mathbf{4.0 \text{ 小时}}$
4. 审批通过请假 **1.0 天**；
5. 发生旷工 **1.0 天**；
6. 审批通过加班 4.0 小时（实际打卡 5.0 小时，按最小值取 **4.0 小时**）。

### 📌 薪资结算推导过程
* **基本工资** = **+￥17,400.00**
* **岗位津贴** = **+￥1,000.00**
* **加班费加发** = $4.0 \text{ 小时} \times 100 \text{ 元/小时} \times 1.5 = \mathbf{+\text{￥}600.00}$
* **离岗综合扣款** = $(2.0 \text{ 离岗小时} + 4.0 \text{ 缺卡小时}) \times 100 \text{ 元/小时} = \mathbf{-\text{￥}600.00}$
* **请假扣款** = $1.0 \text{ 天} \times 800 \text{ 元/天} \times 1.0 = \mathbf{-\text{￥}800.00}$
* **旷工扣款** = $1.0 \text{ 天} \times 800 \text{ 元/天} \times 2.0 = \mathbf{-\text{￥}1,600.00}$

$$\text{最终实发工资} = 17,400 + 1,000 + 600 - 600 - 800 - 1,600 = \mathbf{16,000.00 \text{ 元}}$$
